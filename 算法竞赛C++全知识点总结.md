## 一、竞赛常用C++语法概要
### 1. 基础配置与输入输出
#### （1）万能头文件
竞赛中最常用的头文件，包含所有标准库内容，无需逐个引入：
```cpp
#include <bits/stdc++.h>
using namespace std;
```
> 注意：部分老旧OJ可能不支持，此时可手动引入需要的头文件，如`#include <iostream> #include <vector> #include <algorithm>`等。

#### （2）输入输出优化（必背）
竞赛中`cin/cout`默认同步慢，需关闭同步+解除绑定，或直接用快读快写：
```cpp
// 方法1：优化cin/cout（适合中等数据量）
ios::sync_with_stdio(false);
cin.tie(nullptr);
cout.tie(nullptr);

// 方法2：快读快写（适合1e6+数据量，必背）
int read() {
    int x = 0, f = 1;
    char ch = getchar();
    while (ch < '0' || ch > '9') {
        if (ch == '-') f = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + ch - '0';
        ch = getchar();
    }
    return x * f;
}
void write(int x) {
    if (x < 0) {
        x = -x;
        putchar('-');
    }
    if (x > 9) write(x / 10);
    putchar(x % 10 + '0');
}
// 使用：int a = read(); write(a);
```

#### （3）基础输入输出格式
```cpp
int a, b; cin >> a >> b; // 读两个整数
string s; cin >> s; // 读字符串（不含空格）
getline(cin, s); // 读含空格的整行字符串
printf("%d %lld %.2f", a, b, c); // 格式化输出：整数、长整型、保留2位小数
cout << a << " " << b << endl; // 输出后换行
```

### 2. 核心STL容器与用法（竞赛高频）
| 容器                                          | 适用场景                         | 常用操作                                           | 复杂度                              |
| --------------------------------------------- | -------------------------------- | -------------------------------------------------- | ----------------------------------- |
| `vector<T>`                                   | 动态数组，最常用                 | `push_back()`/`pop_back()`/`size()`/`clear()`/遍历 | 随机访问O(1)，插入删除末尾O(1)      |
| `string`                                      | 字符串处理                       | `size()`/`substr()`/`find()`/`append()`            | 随机访问O(1)，子串操作O(n)          |
| `queue<T>`                                    | 队列（BFS常用）                  | `push()`/`pop()`/`front()`/`back()`                | O(1)                                |
| `stack<T>`                                    | 栈（括号匹配、单调栈）           | `push()`/`pop()`/`top()`                           | O(1)                                |
| `priority_queue<T>`                           | 堆（最值、Dijkstra）             | `push()`/`pop()`/`top()`                           | 插入删除O(log n)                    |
| `set<T>`/`unordered_set<T>`                   | 去重、查找                       | `insert()`/`erase()`/`find()`/`count()`            | set O(log n)，unordered_set平均O(1) |
| `map<key, value>`/`unordered_map<key, value>` | 键值对存储（统计频率、DP记忆化） | `[]`访问/`insert()`/`find()`                       | map O(log n)，unordered_map平均O(1) |
| `pair<T1, T2>`                                | 存储二元组                       | `first`/`second`                                   | -                                   |
| `tuple<T1, T2, T3>`                           | 存储三元组                       | `get<0>()`                                         | -                                   |

#### 高频用法示例
```cpp
// vector去重：先排序再去重
sort(vec.begin(), vec.end());
vec.erase(unique(vec.begin(), vec.end()), vec.end());

// 小顶堆定义（默认是大顶堆）
priority_queue<int, vector<int>, greater<int>> pq;

// map统计频率
unordered_map<int, int> cnt;
cnt[x]++; // 统计x出现的次数

// 遍历vector
for (auto x : vec) cout << x << " ";
for (int i = 0; i < vec.size(); i++) cout << vec[i] << " ";
```

### 4. 高频语法技巧
#### （1）位运算（竞赛高频）
```cpp
int lowbit(int x) { return x & -x; } // 求x最低位的1，如lowbit(6)=2（6的二进制是110）
// 统计二进制中1的个数
int cnt = 0;
while (x) {
    x -= lowbit(x);
    cnt++;
}
// 常用技巧：
// 判断奇偶：x & 1 （奇数返回1，偶数返回0）
// 子集枚举（状压DP必用）：枚举sub的所有子集
for (int s = sub; s; s = (s - 1) & sub) {
    // 处理子集s
}
// 判断第k位是否为1：(x >> k) & 1
```

#### （2）常用类型别名（减少输入）
```cpp
#define ll long long
#define ull unsigned long long
#define pb push_back
#define fi first
#define se second
// 使用：ll a = 1e18; // 等价于long long a = 1e18;
```

#### （3）数据类型范围（避免溢出必记）
| 类型             | 范围                           |
| ---------------- | ------------------------------ |
| `int`            | -2^31 ~ 2^31-1 （约±2e9）      |
| `long long`/`ll` | -2^63 ~ 2^63-1 （约±9e18）     |
| `unsigned int`   | 0 ~ 2^32-1（约4e9）            |
| `double`         | 精度约15位小数，适合浮点数计算 |

> 溢出避坑：两个int相乘可能溢出，需提前转long long：`1LL * a * b`

#### （4）常用算法模板（必背）
```cpp
// 快速幂：求a^b mod p
ll quick_pow(ll a, ll b, ll p) {
    ll res = 1;
    a %= p;
    while (b) {
        if (b & 1) res = res * a % p;
        a = a * a % p;
        b >>= 1;
    }
    return res;
}

// 扩展欧几里得：求ax+by=gcd(a,b)的解，以及乘法逆元
ll exgcd(ll a, ll b, ll &x, ll &y) {
    if (b == 0) { x = 1; y = 0; return a; }
    ll g = exgcd(b, a % b, y, x);
    y -= a / b * x;
    return g;
}
ll mod_inv(ll a, ll p) { // 求a在模质数p下的逆元，a和p互质
    ll x, y;
    exgcd(a, p, x, y);
    return (x % p + p) % p;
}

// 并查集模板
int f[100005];
int find(int x) {
    return f[x] == x ? x : f[x] = find(f[x]); // 路径压缩
}
void merge(int a, int b) {
    int fa = find(a), fb = find(b);
    if (fa != fb) f[fa] = fb;
}
// 初始化：for (int i = 0; i <= n; i++) f[i] = i;
```

| 堆（优先队列）                       | TopK问题、Dijkstra最短路、哈夫曼编码   | O(log n)                       | 默认大顶堆，小顶堆需自定义比较器             |

### 3. 动态规划（DP）
DP核心：**最优子结构 + 无后效性 + 子问题重叠**
#### 高频DP类型
| DP类型 | 适用场景                             | 复杂度           | 典型例题                             |
| ------ | ------------------------------------ | ---------------- | ------------------------------------ |
| 背包DP | 选物品求最值（重量、价值限制）       | O(n*V)           | 01背包、完全背包、多重背包、分组背包 |
| 线性DP | 序列类问题（LIS、LCS、最长公共子串） | O(n²)~O(n log n) | 最长上升子序列、最长公共子序列       |
| 区间DP | 区间类问题（回文串、石子合并）       | O(n³)            | 最长回文子串、最小代价合并石子       |
| 树形DP | 树上问题（选节点、路径和）           | O(n)             | 树的直径、二叉树中的最大路径和       |
| 状压DP | n<=20的排列、子集选择问题            | O(n*2^n)         | 旅行商问题、选子集满足条件           |
| 数位DP | 统计数字区间内的满足条件的数的个数   | O(log n)         | 统计1~n中不含4的数字个数             |
| 计数DP | 统计方案数                           | O(n*m)           | 爬楼梯问题、凑钱币方案数             |

### 7. 其他高频算法

## 三、解题思路与做题方法论
### 核心原则：**先看数据范围，再抽象问题，最后选算法**
#### 步骤1：读题与拆解（5分钟内完成）
1. **先看数据范围**：这是选算法的核心依据，直接决定复杂度上限：
   | 数据范围n | 允许的最坏复杂度 | 可选算法                                       |
   | --------- | ---------------- | ---------------------------------------------- |
   | n <= 10   | O(n!)            | 全排列枚举、DFS暴力                            |
   | n <= 20   | O(2^n)           | 状压DP、子集枚举                               |
   | n <= 100  | O(n³)            | Floyd、区间DP                                  |
   | n <= 1e3  | O(n²)            | DP、排序+二分、Dijkstra（稠密图）              |
   | n <= 1e5  | O(n log n)       | 排序、二分、线段树、Dijkstra（稀疏图）、线性DP |
   | n <= 1e6  | O(n)             | 双指针、单调栈/队列、前缀和、线性筛            |
   | n <= 1e18 | O(log n)         | 快速幂、二分答案、数位DP                       |
2. **理解题意**：手算一遍样例，确认输入输出格式、隐藏条件（如图是否连通、节点编号从0还是1开始、是否需要取模、精度要求）。
3. **确认问题类型**：抽象问题，比如是求最值？计数？路径？匹配？是否存在解？

#### 步骤2：联想知识点与确定算法
根据问题类型和数据范围，联想对应的知识点：
- 看到「区间最值/区间和」→ 线段树/ST表/单调栈/前缀和
- 看到「最短路」→ Dijkstra/SPFA/Floyd
- 看到「最小生成树」→ Prim/Kruskal
- 看到「连通块/分组」→ 并查集/BFS/DFS
- 看到「最长上升子序列」→ 二分优化DP/线性DP
- 看到「背包」→ 背包DP
- 看到「字符串匹配」→ KMP/AC自动机
- 看到「排列组合/计数」→ 组合数学/DP
- 看到「n<=20的选择问题」→ 状压DP
- 看到「树上问题」→ 树形DP/LCA/树链剖分

> 技巧：如果想不到最优算法，先写暴力解法，再尝试优化，至少能拿部分分。

#### 步骤3：编码与边界处理
1. 注意数组开够大小（如n=1e5，数组开n+5，不要开刚好n），全局数组可开更大（如1e6），局部数组不要开太大避免栈溢出。
2. 注意数据类型：大范围整数用long long，乘积运算提前转long long避免溢出。
3. 处理边界情况：n=0、n=1、全0/全1、空字符串等极端场景。
4. 多组输入记得重置全局变量（如并查集的父数组、vis数组）。

#### 步骤4：调试与优化
1. 先跑样例，确认输出和样例一致。
2. 造极端数据测试：比如n=最大值、边界值、特殊构造的数据。
3. 检查常见错误：数组越界、类型溢出、循环边界写错、二分条件写反、图论重边/自环未处理。
4. 优化：如果TLE，先看复杂度是否符合数据范围，再找常数优化（如快读、减少STL不必要的操作）。

## 四、常见坑点与技巧
1. **数组越界**：数组开n+5，不要用下标n，遍历时注意边界。
2. **类型溢出**：两个int相乘可能溢出，用`1LL * a * b`转long long。
3. **多组输入未重置**：每次读入新数据时，重置vector、vis数组、并查集等全局变量。
4. **二分边界错误**：写二分前先明确是左闭右闭还是左闭右开，避免死循环或漏解。
5. **浮点数精度**：计算几何、除法场景不要直接判断相等，用eps判断：`if (abs(a - b) < 1e-6)`。
6. **STL清空陷阱**：vector的`clear()`只清空元素，不释放内存，多次使用时用`vector<T>().swap(vec)`释放内存。
7. **无符号数陷阱**：string的size()是size_t（无符号），不要和int直接比较，比如`for (int i = s.size()-1; i>=0; i--)`会死循环，需先转int：`for (int i = (int)s.size()-1; i>=0; i--)`。
8. **重边与自环**：图论问题记得预处理重边（保留最小权值）、删除自环，避免算法出错。
