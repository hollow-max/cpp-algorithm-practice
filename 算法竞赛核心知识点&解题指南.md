## 一、通用解题流程（拿到题先按这个顺序想）
### 第一步：圈出所有关键信息
✅ **数据范围**：直接决定算法时间复杂度上限（附可过数据规模参考）
| 时间复杂度                                                   | 可过最大数据规模 | 适用场景                 |
| ------------------------------------------------------------ | ---------------- | ------------------------ |
| O(2ⁿ)                                                        | n≤20             | 暴力枚举、状态压缩DP     |
| O(n³)                                                        | n≤500            | 简单DP、Floyd最短路      |
| O(n²logn)                                                    | n≤5000           | 排序+枚举、部分DP优化    |
| O(n²)                                                        | n≤1e4            | 简单DP、稠密图最小生成树 |
| O(nlogn)                                                     | n≤1e5            | 大部分竞赛题的标准要求   |
| O(n)                                                         | n≤1e6            | 线性扫描、单调栈/队列    |
| O(1)                                                         | 任意             | 公式推导、简单模拟       |
| ✅ 特殊要求：是否要求模`1e9+7`、是否要long long、是否多组输入、是否有浮点数精度要求 |                  |                          |
| ✅ 输入输出格式：是否有多余空格、换行、是否有多组测试用例     |                  |                          |
### 第二步：先想暴力解法
不管能不能过，先写能过的暴力代码，至少能拿部分分，再思考优化方向。
### 第三步：匹配知识点
根据题型、数据范围判断属于哪个知识模块，套对应模板/思路。
### 第四步：边界&特殊用例测试
写完代码先自己造样例测试：包括普通样例、边界用例（n=1、全0、全最大值、空集合）、极端用例（所有元素相等、全是质数等），确认无bug再提交。

4. **快读快写模板（处理1e6以上输入输出必用）**
```cpp
// 快读
int read() {
    int x=0, f=1;
    char ch=getchar();
    while(ch<'0' || ch>'9') {
        if(ch=='-') f=-1;
        ch=getchar();
    }
    while(ch>='0' && ch<='9') {
        x=x*10+ch-'0';
        ch=getchar();
    }
    return x*f;
}
// 快写
void write(int x) {
    if(x<0) putchar('-'), x=-x;
    if(x>9) write(x/10);
    putchar(x%10+'0');
}
```

### 2.2 基础数据结构
| 数据结构                 | 时间复杂度                          | 核心操作                            | 典型适用场景                                                 |
| ------------------------ | ----------------------------------- | ----------------------------------- | ------------------------------------------------------------ |
| 单调栈                   | 建栈O(n)，查询O(1)                  | 维护单调递增/递减的栈               | 接雨水、柱状图最大矩形、每个元素左右第一个比它大/小的元素    |
| 单调队列                 | 建队O(n)，查询O(1)                  | 维护滑动窗口的最值                  | 滑动窗口最大值、区间最值查询                                 |
| 链表                     | 插入/删除O(1)，查询O(n)             | 头插/尾插、删除节点                 | 竞赛考得少，多为模拟题                                       |
| 二叉树/二叉搜索树        | 查询/插入/删除O(logn)               | 中序遍历有序                        | 竞赛一般直接用STL的map/set替代                               |
| 平衡树（Treap/Splay）    | 所有操作O(logn)                     | 维护有序序列，支持插入删除查询第k大 | 需要动态维护顺序的场景，竞赛可手写或用pbds库                 |
| 树状数组                 | 单点修改/区间查询O(logn)，空间O(n)  | 前缀和、逆序对、区间和              | 静态/动态前缀和、逆序对、区间求和（只能维护满足结合律的运算） |
| 线段树                   | 区间修改/区间查询O(logn)，空间O(4n) | 懒标记、区间最值/求和/GCD           | 需要区间修改、区间查询的场景，功能比树状数组全               |
| 可持久化线段树（主席树） | 查询O(logn)，空间O(nlogn)           | 维护历史版本，查询区间第k大         | 静态区间第k大、区间不同数个数                                |
| 并查集（DSU）            | 路径压缩+按秩合并后接近O(1)         | 查找根节点、合并集合                | 连通块计数、判断两点是否连通、Kruskal最小生成树              |
| 字典树（Trie）           | 插入/查询O(字符串长度)              | 维护字符串前缀                      | 字符串前缀匹配、单词统计、最大异或和                         |
| 图（邻接表/邻接矩阵）    | 邻接表O(n+m)，邻接矩阵O(n²)         | 建图、遍历                          | 所有图论题的基础                                             |

### 2.3 核心算法
#### 1. 排序算法
| 算法             | 时间复杂度              | 稳定性 | 适用场景                       |
| ---------------- | ----------------------- | ------ | ------------------------------ |
| 快速排序         | 平均O(nlogn)，最坏O(n²) | 不稳定 | 大部分通用排序场景             |
| 归并排序         | O(nlogn)                | 稳定   | 需要稳定排序的场景，逆序对计数 |
| 计数/桶/基数排序 | O(n)                    | 稳定   | 数据范围小、整数排序场景       |
#### 2. 搜索算法
| 算法                | 适用场景                               | 优化技巧                                |
| ------------------- | -------------------------------------- | --------------------------------------- |
| DFS（深度优先搜索） | 枚举所有可能、回溯、树/图的遍历        | 剪枝（可行性/最优性）、记忆化搜索       |
| BFS（广度优先搜索） | 求无权图最短路、层序遍历、状态空间搜索 | 双向BFS（起点终点同时搜，减少搜索空间） |
| 迭代加深搜索        | 搜索深度未知、需要找最小解的DFS        | 限制深度逐层增加，避免DFS爆栈           |
| A*算法              | 带权最短路、启发式搜索                 | 设计启发函数，优先搜索更接近目标的节点  |
#### 3. 动态规划（重中之重，占竞赛题40%以上）
| DP类型     | 核心思路                                                     | 典型例题                                                     |
| ---------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 背包DP     | 状态定义`dp[i][j]`表示前i个物品、容量为j的最优值，转移考虑选/不选当前物品 | 01背包、完全背包、多重背包、分组背包、背包拼接字符串         |
| 线性DP     | 状态按线性顺序转移，通常和序列/路径相关                      | LIS（最长上升子序列）、LCS（最长公共子序列）、最大子段和、打家劫舍 |
| 区间DP     | 状态定义`dp[l][r]`表示区间[l,r]的最优值，转移枚举分割点      | 石子合并、最长回文子序列、矩阵连乘                           |
| 树形DP     | 状态定义在树上，转移考虑子节点的贡献                         | 树的直径、最大独立集、树上最长路径                           |
| 状态压缩DP | 用二进制位表示状态，通常用于n≤20的集合选择问题               | 旅行商问题TSP、状态压缩BFS、插头DP                           |
| 数位DP     | 按数位递推，统计满足条件的数字个数                           | 统计[L,R]中不包含4的数字个数                                 |
| 博弈DP     | 分析博弈状态，通常和SG函数、巴什博弈相关                     | 尼姆博弈、取石子游戏                                         |
#### 4. 贪心算法
✅ 适用场景：问题具有**贪心选择性质**（每一步选局部最优，最终能得到全局最优）
✅ 证明方法：反证法、数学归纳法、交换论证法
✅ 典型例题：区间调度（最多选多少个不重叠区间）、霍夫曼编码、最小生成树（Kruskal）、活动选择问题
#### 5. 字符串算法
| 算法                 | 时间复杂度           | 适用场景                                    |
| -------------------- | -------------------- | ------------------------------------------- |
| 暴力匹配             | O(n*m)               | n,m很小的时候直接用                         |
| KMP                  | O(n+m)               | 单模式串匹配，求模式串在文本串中出现的位置  |
| AC自动机             | O(n+m+匹配次数)      | 多模式串匹配，求文本串中包含多少模式串      |
| 字符串哈希（双哈希） | O(1)查询，O(n)预处理 | 快速判断两个子串是否相等、最长重复子串      |
| Manacher             | O(n)                 | 求最长回文子串长度、回文子串个数            |
| 后缀数组/后缀自动机  | O(nlogn)/O(n)        | 求不同子串个数、最长公共子串、出现k次的子串 |
#### 6. 图论算法
| 算法               | 时间复杂度          | 适用场景                                       |
| ------------------ | ------------------- | ---------------------------------------------- |
| Dijkstra（堆优化） | O(m logn)           | 单源最短路，**不能处理负权边**                 |
| SPFA               | 平均O(m)，最坏O(nm) | 单源最短路，可处理负权边，数据卡得严容易超时   |
| Floyd              | O(n³)               | 多源最短路，n≤500的时候用                      |
| Bellman-Ford       | O(nm)               | 单源最短路，可处理负权边，求负环               |
| Kruskal            | O(mlogm)            | 最小生成树，适合边少的稀疏图                   |
| Prim（堆优化）     | O(mlogn)            | 最小生成树，适合点少的稠密图                   |
| 拓扑排序           | O(n+m)              | 有向无环图的线性排序，判断是否有环、课程表问题 |
| 匈牙利算法         | O(nm)               | 二分图最大匹配                                 |
| KM算法             | O(n³)               | 二分图最大权完美匹配                           |
| Kosaraju/Tarjan    | O(n+m)              | 强连通分量SCC、缩点                            |
| 网络流（Dinic）    | O(m√n)              | 最大流、最小割、二分图匹配                     |
| 费用流             | O(m²n)              | 最小费用最大流、分配问题                       |
#### 7. 其他算法
- 分治：归并排序、快速排序、二分查找、分治求幂、棋盘覆盖
- 模拟：直接按题目要求写代码，注意细节和边界，多为简单题

## 三、常见题型&对应思考路径
### 1. 模拟题
✅ 特征：直接按题目描述的要求模拟流程，没有复杂算法，多为简单题
✅ 思考步骤：
- 拆解题目描述的流程，按步骤写代码
- 重点注意边界：n=0、空输入、数据范围边界、多组输入要清空数组
- 常见坑：输出格式（多余空格/换行）、输入格式（多组数据、空格分割）
✅ 典型题：日期计算、矩阵变换、简单的规则模拟
### 2. 字符串题
✅ 思考步骤：
1. 先看数据范围：n≤100直接暴力匹配
2. 单模式串匹配（一个模式串找文本串）→ KMP
3. 多模式串匹配（多个模式串找文本串）→ AC自动机
4. 问回文相关（最长回文子串、回文个数）→ Manacher/字符串哈希
5. 问子串相关（不同子串个数、最长公共子串）→ 后缀数组/后缀自动机
6. 问前缀匹配 → Trie
### 3. 搜索题
✅ 思考步骤：
1. n≤20 → DFS回溯/状态压缩DP
2. 求无权图最短路/最小步数 → BFS
3. 有约束需要剪枝 → 排序剪枝（先选大的）、可行性剪枝（提前判断不可能）、最优性剪枝（当前路径已经比已知最优差，直接退出）
4. 状态空间大但深度浅 → 双向BFS
5. 需要启发式搜索 → A*算法
### 4. DP题
✅ 思考步骤：
1. 先判断是否有最优子结构、重叠子问题，符合再考虑DP
2. 定义状态：看题目要什么，反推需要记录什么信息（比如背包问题要记录物品序号和容量，所以状态是二维）
3. 找状态转移方程：思考当前状态可以由哪些前置状态转移过来，比如背包问题选/不选当前物品
4. 确定初始状态和边界条件：比如dp[0][j]=0，dp[i][0]=0
5. 优化：空间不够用滚动数组，时间不够用单调队列/线段树优化DP转移
### 5. 数学题
✅ 思考步骤：
1. 先看是否和质数、模、整除相关 → 数论（筛法、逆元、同余）
2. 问计数、排列组合相关 → 组合数学（容斥、卡特兰数、Burnside）
3. 问数字相关（数位、数字个数）→ 数位DP
4. 问期望、概率 → 期望DP/概率公式
5. 问几何相关 → 计算几何（凸包、叉积）
### 6. 图论题
✅ 思考步骤：
1. 先判断是有向图还是无向图，有没有环
2. 求最短路：
   - 没有负权边 → Dijkstra
   - 有负权边且需要求负环 → Bellman-Ford/SPFA
3. 求最小生成树：
   - 边少（m<n²）→ Kruskal
   - 点少（n≤1e3）→ Prim
4. 求连通性/环 → 并查集/拓扑排序/强连通分量
5. 求匹配/流 → 二分图匹配/网络流
### 7. 数据结构题
✅ 思考步骤：
1. 静态区间查询 → 前缀和/差分
2. 动态单点修改+区间查询 → 树状数组/线段树
3. 动态区间修改+区间查询 → 线段树（带懒标记）
4. 维护有序序列+插入删除查询第k大 → 平衡树/Treap
5. 字符串前缀匹配 → Trie
6. 求逆序对 → 归并排序/树状数组

## 四、竞赛常见坑点&避坑指南
1. **数据范围看错**：n是1e5写成O(n²)直接超时，读题先圈数据范围
2. **类型溢出**：两个1e5的数相乘超过int最大值（2e9），一定要用long long，乘法结果记得取模
3. **边界条件漏写**：n=0、空数组、数组下标越界、左右边界开闭写错（二分查找最常考）
4. **浮点数精度问题**：比较两个double不要用`==`，用`fabs(a-b) < 1e-8`，优先用整数运算避免精度
5. **输入输出未优化**：C++没加`ios::sync_with_stdio(false)`，Python没加`input=sys.stdin.readline`，大数据输入直接超时
6. **模运算错误**：`(a*b) % mod`要拆成`((a%mod)*(b%mod))%mod`，负数模要写成`(a%mod + mod) % mod`避免负数
7. **图论坑**：重边、自环没处理，双向边只加了一条，无向图存成有向图
8. **递归坑**：Python默认递归深度1000，DFS超过1e4层要加`sys.setrecursionlimit`，否则栈溢出
9. **STL使用错误**：map的key不存在会插入默认值，判断是否存在用`count()`或`find()`，不要直接访问
10. **输出格式错误**：多了空格、少了换行、多组数据没清空数组导致上一组数据影响下一组

## 五、刷题&备赛建议
1. **刷题顺序**：先刷基础题（洛谷入门题、蓝桥杯真题简单题）→ 刷专题（DP、图论、字符串、数学各刷20-30题）→ 刷历年竞赛真题（ACM区域赛、NOIP、蓝桥杯省赛/国赛）
2. **模板整理**：提前整理好快读快写、常用算法模板（Dijkstra、KMP、欧拉筛、DP模板等），比赛的时候直接复制，不用现场写
3. **错题本**：记录做错的题的错误原因、对应知识点，反复复习
4. **时间管理**：竞赛一般3-5小时，先做简单题拿基础分，一道题超过30分钟没思路直接跳，不要死磕
